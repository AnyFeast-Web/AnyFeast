# AnyFeast Backend Architecture & Schema Overview

This document provides a high-level overview of the AnyFeast platform's backend infrastructure. It outlines the structure of the cloud database (Firestore) and the primary application programming interfaces (FastAPI) responsible for driving data to both the Web Portal and the mobile Flutter App.

---

## Part 1: Database Schema (Firestore Collections)

AnyFeast utilizes Google Cloud Firestore as its primary NoSQL document database. Data is organized into independent collections.

### 1. `users` (System Administrators & Nutritionists)
*Purpose: Maps directly to Firebase Authentication to manage nutritionist staff profiles.*
* **uid** (`String`): Unique identifier matching Firebase Auth.
* **email** (`String`): User's login email.
* **name** (`String`): User's full display name.
* **role** (`String`): Authorization level (e.g., `"admin"`, `"nutritionist"`).
* **created_at** (`Timestamp`): Account creation date.

### 2. `clients` (End-Consumers / Patients)
*Purpose: Stores external client profiles managed by the nutritionists.*
* **id** (`String`): Unique document ID.
* **nutritionist_id** (`String`): Reference to the assigned `users.uid`.
* **status** (`String`): e.g., `"active"`, `"inactive"`.
* **personal_info** (`Map`): Nested map capturing demographics.
  * **first_name** (`String`)
  * **last_name** (`String`)
  * **email** (`String`)
  * **phone** (`String`)
  * **dob** (`Timestamp` / `Date`)
* **goals** (`Array<String>`): e.g., `["fat_loss", "muscle_gain"]`.
* **measurements** (`Map`): Snapshot of height, weight, activity multiplier.
* **created_at**, **updated_at** (`Timestamp`)

### 3. `meal_plans`
*Purpose: Stores generated 7-day nutritional schedules assigned to clients.*
* **id** (`String`): Unique document ID.
* **client_id** (`String`): Reference to the `clients` collection.
* **nutritionist_id** (`String`): Creator reference.
* **title** (`String`): e.g., `"Cutting Phase Plan - W1"`.
* **status** (`String`): e.g., `"draft"`, `"active"`, `"expired"`.
* **date_range** (`Map`): `start_date` and `end_date` (`Timestamps`).
* **grid** (`Map`): The core 7-day schedule map.
  * *Keys:* Days (e.g., `"monday"`, `"tuesday"`)
  * *Values:* Nested maps of Meal Types (e.g., `"breakfast"`, `"lunch"`) containing an Array of `MacroMeal` objects:
    * **name** (`String`)
    * **calories** (`Number`)
    * **protein_g**, **carbs_g**, **fat_g** (`Number`)
* **total_nutrition_targets** (`Map`): Recommended macro targets for calculating compliance.

### 4. `consultations`
*Purpose: Archives chat history and structured clinical assessments.*
* **id** (`String`): Unique document ID.
* **client_id** (`String`): Reference to the `clients` collection.
* **nutritionist_id** (`String`): Creator reference.
* **type** (`String`): `"structured"` or `"chat"`.
* **status** (`String`): e.g., `"draft"`, `"completed"`.
* **form_data** (`Map`): For structured sessions (Captures diet adherence, sleep metrics).
* **blood_markers** (`Array<Map>`): Laboratory results and automated warnings.
* **summary_notes** (`String`): Autogenerated or manual summary.
* **created_at**, **updated_at** (`Timestamp`)

### 5. `ingredients`
*Purpose: The global nutrition intelligence database.*
* **id** (`String`): Unique document ID.
* **name** (`String`): e.g., `"Raw Avocado"`.
* **category** (`String`): e.g., `"fats"`, `"protein"`.
* **source** (`String`): Data origin, e.g., `"usda"`.
* **macros_per_100g** (`Map`):
  * **calories** (`Number`)
  * **protein_g** (`Number`)
  * **carbs_g** (`Number`)
  * **fat_g** (`Number`)
  * **fiber_g** (`Number`)

## Part 2: REST API Endpoints (FastAPI)

The Python FastAPI backend serves as the secure middleware between the client applications (Web/Flutter) and the Firestore database.

> **Security Note:** All endpoints below require a valid **Firebase Bearer JWT** passed in the `Authorization` header.

### Client Management API
*   `GET /api/v1/clients`
    *   **Purpose:** Retrieve a paginated list of all active clients assigned to the requesting nutritionist.
*   `POST /api/v1/clients`
    *   **Purpose:** Create and onboard a new client profile.
*   `GET /api/v1/clients/{client_id}`
    *   **Purpose:** Fetch the full profile, historical measurements, and settings for a specific client.
*   `PUT /api/v1/clients/{client_id}`
    *   **Purpose:** Update client demographics or goal targets.

### Meal Plan Generator API
*   `GET /api/v1/mealplans`
    *   **Purpose:** Retrieve all meal plans (filtered by client or status).
*   `POST /api/v1/mealplans`
    *   **Purpose:** Save a new 7-day meal plan grid into the database.
*   `GET /api/v1/mealplans/{plan_id}`
    *   **Purpose:** Load a specific meal plan into the builder grid.
*   `PUT /api/v1/mealplans/{plan_id}`
    *   **Purpose:** Update/Edit an existing meal plan (creates a new version/draft).

### Consultation Archives API
*   `GET /api/v1/consultations`
    *   **Purpose:** Retrieve consultation history for the global index or specifically scoped to one client.
*   `POST /api/v1/consultations`
    *   **Purpose:** Create a new structured consultation session or append a chat thread.
*   `GET /api/v1/consultations/{consultation_id}`
    *   **Purpose:** Load historic consultation data, including flagged blood markers and PDF summaries.
*   `PUT /api/v1/consultations/{consultation_id}`
    *   **Purpose:** Auto-save ongoing consultation drafts.

### Nutrition Database API
*   `GET /api/v1/ingredients`
    *   **Purpose:** Perform a global search with query parameters (`?search=avocado&category=fats`) to retrieve macros.
*   `POST /api/v1/ingredients`
    *   **Purpose:** Allow admins to supplement the database with custom or proprietary food items.
*   `PUT /api/v1/ingredients/{ingredient_id}`
    *   **Purpose:** Patch or inline-edit macro data for an existing ingredient.

### Integrations & Webhooks (n8n/Automations)
*   `POST /api/v1/webhooks/trigger-order`
    *   **Purpose:** Fires the n8n automation pipeline to extract a meal plan's ingredient list and dispatch a fulfillment order via external APIs (e.g., AnyFeast kitchens/vendors).
*   `POST /api/v1/webhooks/dispatch-email`
    *   **Purpose:** Generates a server-side PDF of a particular meal plan/consultation and triggers SendGrid to deliver it to the client.
